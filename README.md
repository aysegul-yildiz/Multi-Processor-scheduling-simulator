# Multi-Processor-scheduling-simulator
This project was developed for CS 342, operating systems course with my teammate Sena Beyza Ural. This program simulates a multi processor environment where threads represent cpu cores. It supports both single queue and multi queue scheduling architectures to analyze performance trade offs in task turnaround times.  
Implements FCFS (first come first served) and SJF (shortest job first).  
mschedule.c: main simulation logic and thread management  
makefile: builds script for the project.  
report: experimental analysis of turnaround times across different configurations
